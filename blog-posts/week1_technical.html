Week 2 Post - Class vs. ID in CSS

HTML elements can be modified via CSS attributes on their own, but sometimes you need to be a little more specific. In many cases, both a "class" element and an "ID" element would get the job done, but certain situations call for one over the other. A good analogy I've found on the web for thinking about classes versus ID's is to imagine an iPod at Best Buy. All the iPods of similar type, color and size will have the same barcode. This tells the store that it is a specific "class" of iPod. However, if you wanted to refer that ONLY that iPod you would need its serial number. This is like an ID - a unique modifier that refers to only a specific element. If you want to change a number of elements at once, use a class.
